# Project Planning Questions

Use these questions to gather requirements before generating PRD and team structure.

## Core Questions

### 1. Project Overview
- What are you building? (brief description)
- What problem does it solve?
- Who is the target user?

### 2. Technical Requirements
- What tech stack do you prefer? (or should I suggest?)
- Any existing codebase to work with?
- What platforms? (web, mobile, desktop, API)

### 3. Scope & Features
- What are the must-have features for v1?
- Any nice-to-have features?
- What's out of scope?

### 4. Quality & Constraints
- How important is test coverage? (high/medium/low)
- Any performance requirements?
- Security considerations?
- Timeline expectations?

### 5. Team Preferences
- Preferred team size? (minimal, balanced, comprehensive)
- Any specific roles you want? (QA, DevOps, etc.)
- Should agents work autonomously or check in frequently?

## Team Size Guidelines

**Minimal (2 agents)**
- PM + Developer
- Good for: small projects, prototypes, solo oversight

**Balanced (3-4 agents)**
- PM + Developer + QA
- Or: PM + Frontend Dev + Backend Dev
- Good for: medium projects, quality-focused work

**Comprehensive (5+ agents)**
- PM + Multiple Devs + QA + DevOps
- Good for: large projects, complex systems
